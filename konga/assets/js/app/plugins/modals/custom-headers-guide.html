Header configurations

Key Description
Header Name/Title The title for header, may contain {*} placeholder when iterate thru claims, take a look at iterate field below. Example: http-kong-id-token
Header Value the expression, define the value in the environment, for example userinfo.email For OpenID Connect use case the environment is populated with 2 root values id_token and userinfo. If you not use environment values then plugin will proceed with the given values. Example:
  • Environment values: userinfo, userinfo.email, id_token, id_token.exp
  • Any Custom vaues: 123, abcd, version 4, etc...
Header Format the format used to encode the value. Formats: string(as it is), base64, urlencoded, list, and jwt(with none alg)
Separator it is use when your header format is list type. It join the list of values by separator. Example: , (comma)
Iterate the header value should point to an environment values, for example id_token It use to iterate thru keys/values and create separate header for every key. The captured key is accessible in header name via special placeholder i.e. {*}.

Examples:

Header Name/Title Value Format Separator Iterate
kong-openidc-id-token id_token [jwt | base64] No
kong-openidc-userinfo userinfo [jwt | base64] No
kong-openidc-id-token-{*} id_token [string | urlencoded | base64] Yes
kong-openidc-userinfo-{*} userinfo [string | urlencoded | base64] Yes
kong-openidc-userinfo-email userinfo.email [string | urlencoded | base64] No
kong-openidc-id-token-exp id_token.exp [string | base64] No
kong-version version 1.3 [string | urlencoded | base64] No
kong-userinfo-roles userinfo.roles list
